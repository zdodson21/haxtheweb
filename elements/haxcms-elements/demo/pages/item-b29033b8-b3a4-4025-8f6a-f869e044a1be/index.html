HAX11ty is a merger between HAX and 11ty. It uses these technologies in the following way:

  • Users edit .md files locally and make modifications in /src/ directory
  • hax11ty watches for changes and rebuilds files w/ 11ty, refreshing the screenâ€‹ (Make need to have Network inspect w/ cache disabled due to rapid changes and service worker generation)
  • this is compiled to a valid HAXcms data package to be used either completely static

Reasons to use this

  • You like writing in markdown and like 11ty workflow
  • You don't want to think about theme or setup and want to leverage the HAX block ecosystem
  • You want to hit higher lighthouse scores on Google Page Speed to improve response times in low performance environments
